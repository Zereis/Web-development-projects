Skillnaden mellan inline css och externa stillmallar

CSS skrivs i externa stilmallar som sen sparas till CSS-filer. Det finns också så kallad Inline Styling som man ibland använder sig av för att påverka en HTML sidas utseende. Där skriver man ut vilken stil man vill att sidan ska ha, till exempel vilken färg, typsnitt och bredd olika element ska ha.

inline css

Fördelar

  • Du kan enkelt och snabbt infoga CSS-regler på en HTML-sida. Det är därför den här metoden är användbar för att testa eller förhandsgranska ändringarna och utföra snabbkorrigeringar på din webbplats
  • Du behöver inte skapa och ladda upp ett separat dokument som i den externa stilen.

Nackdelar

  • Eftersom CSS-koden finns i ett separat dokument kommer dina HTML-filer att ha en renare struktur och mindre i storlek.
  • Du kan använda samma .css-fil för flera sidor.
external css

Fördelar

  • Eftersom CSS-koden finns i ett separat dokument kommer dina HTML-filer att ha en renare struktur och mindre i storlek.
  • Du kan använda samma .css-fil för flera sidor.

Nackdelar

  • Att lägga till CSS-regler till varje HTML-element är tidskrävande och gör din HTML-struktur rörig.
  • Att styla flera element kan påverka din sidas storlek och nedladdningstid.
